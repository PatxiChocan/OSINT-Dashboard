# Aletheia Dashboard — Instalación en Entorno Compartido (Red de Semantic)

**Versión:** 1.0  
**Fecha:** 2026-05-18  
**Clasificación:** Interno

---

## 1. Objetivo

Este documento describe los pasos para desplegar Aletheia Dashboard en un entorno compartido de la Red de Semantic, donde múltiples usuarios (analistas, administradores, clientes) acceden desde diferentes máquinas a través de la red interna.

---

## 3. Configuración de Red

### 3.1 IP del servidor en la Red de Semantic

Asignar una IP fija al servidor Aletheia dentro del rango de la red compartida. Ejemplo:

```
IP servidor: 192.168.10.50
Máscara:     255.255.255.0
Gateway:     192.168.10.1
DNS:         192.168.10.1 (o 8.8.8.8)
```

### 3.2 Ajustar Nginx para escuchar en la IP de red

Editar `/home/kali/Aletheia-Dashboard/nginx-aletheia.conf`:

```nginx
server {
    listen 443 ssl;
    server_name 192.168.10.50;   # IP de la máquina en la red Semantic
                                  # o el hostname: aletheia.semantic.local

    ssl_certificate     /etc/nginx/ssl/aletheia.crt;
    ssl_certificate_key /etc/nginx/ssl/aletheia.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    client_max_body_size 20M;

    location / {
        proxy_pass         http://127.0.0.1:5000;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto https;
        proxy_read_timeout 300s;
    }
}

server {
    listen 80;
    server_name 192.168.10.50;
    return 301 https://$host$request_uri;
}
```

### 3.3 Generar certificado TLS para la red interna

```bash
sudo mkdir -p /etc/nginx/ssl

# Opción A: Certificado auto-firmado (para entornos de prueba)
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/aletheia.key \
  -out /etc/nginx/ssl/aletheia.crt \
  -subj "/CN=192.168.10.50/O=Semantic/C=ES" \
  -addext "subjectAltName=IP:192.168.10.50,DNS:aletheia.semantic.local"

# Opción B: Certificado firmado por la CA interna de Semantic
# (Solicitar al administrador de PKI el cert firmado y copiarlo en /etc/nginx/ssl/)
```

### 3.4 Firewall — puertos a abrir en el servidor

```bash
# En el servidor (ufw o iptables):
sudo ufw allow 443/tcp    # HTTPS — acceso de clientes de la red
sudo ufw allow 80/tcp     # HTTP — redirige a HTTPS
sudo ufw allow 8080/tcp   # Keycloak — solo si los clientes necesitan acceso directo
# NO exponer 5000 (Gunicorn) ni 5432 (PostgreSQL) fuera del servidor
```

## 4. Instalación Paso a Paso

### 4.1 Clonar el repositorio

```bash
git clone <url-repositorio> /home/kali/Aletheia-Dashboard
cd /home/kali/Aletheia-Dashboard
```

### 4.2 Crear entorno virtual e instalar dependencias

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Dependencias adicionales no en requirements.txt
pip install flask-migrate authlib psycopg2-binary python-dotenv stix2-validator
```

### 4.3 Configurar variables de entorno

Crear el fichero `.env` en la raíz del proyecto:

```bash
cat > /home/kali/Aletheia-Dashboard/.env << 'EOF'
# Base de datos
DATABASE_URL=postgresql://aletheia:aletheia@localhost:5432/aletheia_db

# Flask
SECRET_KEY=cambiar-por-secreto-aleatorio-seguro

# APIs OSINT
SHODAN_API_KEY=tu_clave_shodan
VIRUSTOTAL_API_KEY=tu_clave_virustotal
INTELX_API_KEY=tu_clave_intelx
ABUSEIPDB_KEY=tu_clave_abuseipdb
URLSCAN_API_KEY=tu_clave_urlscan
GITHUB_TOKEN=tu_github_token

# MISP (opcional)
MISP_URL=https://misp.semantic.local
MISP_API_KEY=tu_clave_misp
EOF
```

> **Importante:** Añadir `.env` a `.gitignore` para no incluirlo en el repositorio.

### 4.4 Configurar la base de datos PostgreSQL

```bash
# Crear usuario y base de datos
sudo -u postgres psql << 'EOF'
CREATE USER aletheia WITH PASSWORD 'aletheia';
CREATE DATABASE aletheia_db OWNER aletheia;
GRANT ALL PRIVILEGES ON DATABASE aletheia_db TO aletheia;
EOF

# Aplicar migraciones
cd /home/kali/Aletheia-Dashboard
source venv/bin/activate
export FLASK_APP=run.py
flask db upgrade
```

### 4.5 Arrancar Keycloak y OpenLDAP (Docker)

```bash
cd /home/kali/Aletheia-Dashboard/docker
docker compose up -d

# Verificar que están corriendo
docker compose ps
```

Salida esperada:
```
aletheia-keycloak    Up   0.0.0.0:8080->8080/tcp
aletheia-ldap        Up   0.0.0.0:389->389/tcp
aletheia-ldapadmin   Up   0.0.0.0:8090->80/tcp
```

### 4.6 Configurar el Realm en Keycloak

Acceder a `http://192.168.10.50:8080` → Admin Console (admin / admin1234):

1. **Crear Realm:** `aletheia`
2. **Crear Client:** `aletheia-app`
   - Client authentication: ON
   - Valid redirect URIs: `https://192.168.10.50/auth/callback`
   - Web origins: `https://192.168.10.50`
3. **Crear Roles de Realm:** `admin`, `analyst`, `client`
4. **Crear usuarios** y asignarles el rol correspondiente
5. Copiar el **Client Secret** desde la pestaña Credentials del client `aletheia-app`

Actualizar `app/__init__.py` con los datos del entorno:

```python
KEYCLOAK_URL      = "http://localhost:8080"         # Keycloak escucha en el servidor
KEYCLOAK_REALM    = "aletheia"
KEYCLOAK_CLIENT   = "aletheia-app"
KEYCLOAK_SECRET   = "<client-secret-copiado>"
KEYCLOAK_REDIRECT = "https://192.168.10.50/auth/callback"
```

### 4.7 Instalar y configurar Nginx

```bash
# Copiar la configuración
sudo cp /home/kali/Aletheia-Dashboard/nginx-aletheia.conf \
        /etc/nginx/sites-available/aletheia

sudo ln -s /etc/nginx/sites-available/aletheia \
           /etc/nginx/sites-enabled/aletheia

# Deshabilitar el site default si interfiere
sudo rm -f /etc/nginx/sites-enabled/default

# Verificar configuración
sudo nginx -t

# Recargar Nginx
sudo systemctl reload nginx
sudo systemctl enable nginx
```

### 4.8 Iniciar Gunicorn

```bash
cd /home/kali/Aletheia-Dashboard
source venv/bin/activate

# Arranque manual (para pruebas)
gunicorn -w 4 -b 127.0.0.1:5000 run:app

# Para producción — crear un servicio systemd (ver sección 5)
```

## 5. Servicio systemd (Producción)

Crear el fichero de servicio para que Gunicorn arranque automáticamente:

```bash
sudo tee /etc/systemd/system/aletheia.service << 'EOF'
[Unit]
Description=Aletheia Dashboard (Gunicorn)
After=network.target postgresql.service docker.service
Wants=docker.service

[Service]
User=kali
Group=kali
WorkingDirectory=/home/kali/Aletheia-Dashboard
EnvironmentFile=/home/kali/Aletheia-Dashboard/.env
ExecStart=/home/kali/Aletheia-Dashboard/venv/bin/gunicorn \
          -w 4 \
          -b 127.0.0.1:5000 \
          --timeout 120 \
          --access-logfile /home/kali/Aletheia-Dashboard/logs/access.log \
          --error-logfile /home/kali/Aletheia-Dashboard/logs/error.log \
          run:app
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
EOF

# Crear directorio de logs si no existe
mkdir -p /home/kali/Aletheia-Dashboard/logs

# Activar y arrancar el servicio
sudo systemctl daemon-reload
sudo systemctl enable aletheia
sudo systemctl start aletheia
sudo systemctl status aletheia
```

## 6. Acceso desde los Clientes de la Red

### 6.1 Configurar el DNS interno (opcional pero recomendado)

En el servidor DNS de la Red de Semantic, añadir el registro:

```
aletheia.semantic.local  →  192.168.10.50
```

### 6.2 Instalar el certificado en los navegadores de los clientes

Si se usa un certificado auto-firmado, los usuarios deberán aceptar la excepción de seguridad o instalar el certificado en su almacén de confianza.

**Windows:** Doble clic en `aletheia.crt` → Instalar → "Entidades de certificación raíz de confianza"

**Linux:**
```bash
sudo cp aletheia.crt /usr/local/share/ca-certificates/aletheia.crt
sudo update-ca-certificates
```

**macOS:** Doble clic → Llavero → "Siempre confiar"

### 6.3 URL de acceso

```
https://192.168.10.50
# o con DNS configurado:
https://aletheia.semantic.local
```

## 9. Solución de Problemas Comunes

| Síntoma | Causa probable | Solución |
|---------|---------------|----------|
| Login redirige a Keycloak pero vuelve con error | `KEYCLOAK_REDIRECT` no coincide con el Valid redirect URI configurado en Keycloak | Verificar que la URL de callback en `__init__.py` y en Keycloak son idénticas |
| Error 502 Bad Gateway en Nginx | Gunicorn no está corriendo | `sudo systemctl restart aletheia` |
| `SESSION_COOKIE_SECURE` warnings | Configuración de cookie en entorno HTTP/HTTPS mixto | Ya está configurado como `False`; no modificar |
| Docker containers no arrancan | Puerto 8080 ocupado | `sudo lsof -i :8080` y matar el proceso conflictivo |
| `psycopg2` import error | Falta el driver PostgreSQL para Python | `pip install psycopg2-binary` |
| Herramienta CLI no encontrada | No está en PATH | Instalarla y verificar con `which <herramienta>` |
| Certificado no confiable en clientes | Cert auto-firmado | Instalar el cert en los clientes o usar una CA interna |
